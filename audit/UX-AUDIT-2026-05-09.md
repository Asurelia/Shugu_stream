## 2. Défauts critiques (P0)

### Sécurité auth — PR #130 incomplète

1. **`/auth/me` operator-only** (`backend/shugu/routes/auth.py:354`) : `require_operator` + `is_operator=True` hardcodé. Conséquence : un membre régulier connecté via `/account/login` (cookie `shugu_user_access` posé), puis qui navigue vers `/`, voit son `fetchAuthStatus()` (`frontend/src/app/_client.tsx:170`) renvoyer 401. Le HUD redessine "Log in / Sign up". **Le membre est invisiblement déloggué sur la home.** La PR #130 unifie le formulaire, pas l'état.

2. **`/auth/logout` n'efface pas les cookies user** (`backend/shugu/routes/auth.py:123`) : `_clear_cookies()` ne supprime que `ACCESS_COOKIE`/`REFRESH_COOKIE`, jamais `USER_ACCESS_COOKIE`/`USER_REFRESH_COOKIE`. Fuite de session.

### Mobile cassé

3. **Aucun `width=device-width`** dans le viewport (`frontend/src/app/layout.tsx:95`), aucun breakpoint `md:`/`sm:` dans `ViewerStage.tsx`, `VisitorLogin.tsx:11` hardcode `right-[380px]`. **L'app est de fait desktop-only.**

### Viewer fragile

4. **`LoadingScreen` infini en cas d'erreur VRM** (`frontend/src/components/vrmViewer.tsx:33-36`) : pas de progress, pas de catch visuel, juste `console.error`. Sur 28 MB en connexion lente, l'utilisateur ne sait pas si ça charge ou si c'est cassé.
5. **Aucun fallback WebGL** : pas de détection `WebGLRenderingContext`, pas de `webglcontextlost` listener.

### Design System bypass

6. **Page Assets entièrement en inline-styles** (`frontend/src/app/[username]/admin/assets/_client.tsx:213-345`) — zéro `Glass*`, `confirm()` natif blanc OS au milieu du glassmorphism.
7. **`AdminModal` legacy** (`frontend/src/components/AdminModal.tsx:71`) sans `role="dialog"`, sans Esc, sans focus trap, bouton fermeture sans `aria-label`.

## 3. Différences vs applications standards

### Anti-patterns à corriger

1. Iconographie en caractères Unicode mystiques (◉ ◈ ⌬ ⊛ ◇ ✦ ☼) au lieu de Lucide/Heroicons
2. Vocabulaire mystique pour actions admin ("tissés au veil", "♡" pour ban, "✦" pour Promouvoir VIP)
3. Mobile en rail horizontal scrollable au lieu de bottom-tab nav
4. Sidebar à plat sans hiérarchie (11 items siblings)
5. Boutons OAuth fake (Google/Discord/Twitch) sans backend
6. `/[username]/account` mockée avec faux Visa, factures `INV-0925`, `tier="vip"` hardcodé
7. VIP room expose `LIVEKIT_API_KEY` au end-user
8. CTA "Start Stream" décoratif (pas de `onClick`)
9. Drag-drop Schedule annoncé mais non implémenté
10. Mocks partout sans pill "MOCK"

### Choix créatifs intentionnels (à documenter)

11. Creator Home en mode "design comparator" en prod (6 variations)
12. IDE-style Scene Editor vs onglets simples (deux modèles mentaux dans le même produit)
13. Réactions emoji depuis l'audio (pas le chat) — inverse du pattern Twitch
14. Caméra figée + idle wave 5 mm (pas signalé visuellement)
15. Brand-menu en pill collapsible (pattern jamais vu sur Twitch/YouTube)
16. Pas de "browser source" mode (inverse du standard VTuber)

### Manques par rapport au standard streaming

17. Pas de page "watch" type Twitch (`/[creator]` n'existe que pour l'admin)
18. Pas de bouton "Suivre" entre visiteur anonyme et VIP 9 €
19. Pas de planning/VOD/clips/chat-only/slow-mode/emote-only
20. Visiteur anonyme ne voit PAS Shugu écrire (filtre `_client.tsx:122`) — barrière a11y dure pour les sourds
21. Sub-goal présent en code mais NON RENDU (dead code)

## 7. Workflow d'orchestration

**Approche ruflo-driven** : `ruflo-autopilot:autopilot-coordinator` exécute le cycle refactor → tests → commit → PR → CI → merge. Chaque PR isolée dans une worktree dédiée pour éviter les conflits.

**Discipline maintenue** :
- Modulaire transversal : modules ≤400 lignes, 1 responsabilité, JSDoc FR
- Sonnet 4.6 implémentation + review, Haiku 4.5 mécanique
- Verify before completion (diff + CI verte avant merge)
- Pas de bypass hooks, pas de force push, pas d'amend après PR ouverte
